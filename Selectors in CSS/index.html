CSS Selector

CSS selectors are used to find or select the HTML elements we want to style. (or)
CSS selectors are patterns used to select and style HTML elements. They tell the browser which element(s) the CSS rules should apply to.

Types of Selectors in CSS:

CSS selectors are mainly divided in five categories, they are:-

  • Simple Selectors: Select elements by their name, id, or class.
  • Combinator Selectors: Select elements based on their relationship with other elements (like inside, next to, etc).
  • Pseudo-class Selectors: Select elements in a special state (like hover, first child, etc).
  • Pseudo-element Selectors: Select and style part of an element (like first letter, first line, etc).
  • Attribute Selectors: Select elements using their attributes or attribute values (like [type="text"]).

1. Simple Selectors:

Simple selectors are the most basic CSS selectors that are used to select HTML elements directly without considering their relationship with other elements.
They target elements based on their name, id, class, or attributes.

  • ID Selector:
  • Class Selector:
  • Tag Name/ Element Selectors:

    • 1. ID Selectors:-

    The ID selector in CSS is used to style a specific element by targeting its unique id attribute.
    Since an ID must be unique within a page, it is used to select only one element.
    To use an ID selector, write a hash symbol (#) followed by the ID name.

    Example: #header { color: blue; }

    2. Class Selector:-

    The class selector uses the class attribute of HTML elements to select one or multiple elements.
    Unlike IDs, classes are not unique and can be applied to many elements on the same page.
    To select elements with a specific class, write a dot (.) character, followed by the class name.

    Example: .para { color:black; }

    3. Tag Name Selector:-

    The tag name selector (also called the type selector) is used to select all elements of a given HTML tag.
    It applies styles to every occurrence of that tag in the document.
    To use it, simply write the name of the HTML tag without any symbol.

    Example: p { color: black; }

    2. Combinator Selectors:

    Combinator selectors in CSS are used to explain the relationship between two or more elements.
    They help in selecting elements based on how they are placed inside or next to each other in HTML.
    There are four main combinators in CSS:

  • Descendant combinator (space):
  • Child combinator (>):
  • Next sibling combinator (+):
  • Subsequent-sibling combinator (~):

    • 1. Descendant combinator (space):-

    The descendant combinator (a single space) is used to select elements that are nested inside another element, no matter how deep they are (children, grandchildren, etc).

    Example: The following CSS selects all <p> elements that are inside <div> elements.

                <!-- HTML Code -->
            <div>

                <p>This paragraph is inside div (will be blue & bold).</p>

                <section>
                    <p>This paragraph is inside a section inside div (also blue & bold).</p>
                </section>

            </div>

            <!-- CSS Code -->
            div p {
                color: blue;
                font-weight: bold;
            }

    2. Child combinator (>):-

    The child combinator selects all elements that are direct children of a specified element.

    The following example selects all elements that are direct children of <div>.

    Example: 

                <!-- HTML Code -->
            <div>

                <p>This paragraph is a DIRECT child of div (will be yellow).</p>

                <section>
                    <p>This paragraph is inside section, NOT a direct child of div (not yellow).</p>
                </section>

            </div>
            
            <!-- CSS Code -->
            div > p {
                background-color: yellow;
                padding: 5px;
                border: 1px solid black;
            }

    3. Adjacent / Next Sibling Combinator (+):-

    Adjacent Sibling Selector selects an element that is the immediately next sibling of another element.
    OR
    The next sibling combinator is used to select an element that is directly after a specific element.
    Sibling elements must have the same parent element.

    The following example selects the first <p> element that immediately follows a <div>, and share the same parent:

    Example: 

                <!-- HTML Code -->
            <div>
            
                <p>This paragraph is a DIRECT child of div (will be yellow).</p>
                
                <section>
                    <p>This paragraph is inside a section, not a direct child of div (not yellow).</p>
                </section>

            </div>
            
            <!-- CSS Code -->
            div > p { 
                background-color: yellow;
                padding: 10px;
                border: 1px solid black;
            }

    4. Subsequent-sibling Combinator (~):-

    The ~ combinator selects all elements that are next siblings of a specified element.
    The following example selects all p elements that are next siblings of div, and share the same parent: 

                <!-- Html Code -->
            <div>
                <p>This is Paragraph One.</p>    
            </div>

            <p>This is Paragraph Two.</p> 

            <Span>Some other line of code.</Span>    

            <p>This is Paragraph Three.</p>        
            
            <p>!-- CSS Code --</p>   
            
            div ~ p {
                background-color: red;
                padding: 10px;
                border: 2px solid black;
            }

    3. Pseudo-class Selectors:-

    A pseudo-class selector is used to define a special state of an element. It is written with a colon (:) followed by the pseudo-class name.
    They are useful for styling elements when a user hovers, clicks, focuses, or when an element is in a particular position (like first or last child). Some common use for pseudo-classes:

  • Style an element when a user moves the mouse over it
  • Style visited and unvisited links differently
  • Style an element when it gets focus
  • Style valid/invalid/required/optional form elements
  • Style an element that is the first child of its parent

    • Types of Pseudo-class Selectors:-

  • 1. Link-related pseudo-classes:
  • 2. Form-related pseudo-classes:
  • 3. Structural pseudo-classes:
  • 4. UI/State pseudo-classes:

    • 1. Link-related pseudo-classes:-

    Link-related pseudo-classes are special CSS selectors used to style hyperlink () elements based on their state - whether the link is unvisited, visited, being hovered over, or currently clicked.
    These pseudo-classes allow you to change the appearance of links dynamically depending on the user interaction.
    Link-related Pseudo-classes & Their Meaning:-

  • :link:- Selects links that have not been visited yet.
  • :visited:- Selects links that the user has already visited.
  • :hover:- Selects a link when the mouse pointer is over it.
  • :active:- Selects a link at the moment it is being clicked.

    • 2. Form-related pseudo-classes:-

    Form-related pseudo-classes in CSS are used to style form elements (input, textarea, select, etc.)
    based on their state or condition - like when a field is focused, disabled, checked, required, valid, or invalid.
    They help make forms more interactive, user-friendly, and clear for users. Form-related Pseudo-classes & Their Meaning:-

  • :focus:- Selects an element when it is focused (clicked or typing inside).
  • :disabled:- Selects disabled form fields that cannot be interacted with.
  • :enabled:- Selects enabled form fields that can be used.
  • :checked:- Selects radio buttons / checkboxes that are checked.
  • :required:- Selects input fields with the required attribute.
  • :optional:- Selects input fields that are not required.
  • :valid:- Selects inputs where the entered value is valid.
  • :invalid:- Selects inputs where the entered value is invalid.

    • 3. Structural Pseudo-classes:-

    Structural pseudo-classes in CSS are used to style elements based on their <strong>position or structure</strong> within the parent element (like first child, last child, nth child, etc.).
    They allow us to apply styles without adding extra classes or IDs, making the code cleaner and more efficient. Structural Pseudo-classes & Their Meaning:-

  • :first-child:- Selects an element if it is the first child of its parent.
  • :last-child:- Selects an element if it is the last child of its parent.
  • :nth-child(n):- Selects elements based on their position (e.g., 2nd, 3rd, every even/odd element).
  • :nth-last-child(n):- Selects elements based on their position counting from the last child.
  • :only-child:- Selects an element if it is the only child of its parent.
  • :first-of-type:- Selects the first element of a particular type (e.g., first <p> inside parent).
  • :last-of-type:- Selects the last element of a particular type inside the parent.
  • :nth-of-type(n):- Selects elements of a specific type based on their order.
  • :nth-last-of-type(n):- Selects elements of a specific type counting from the last.
  • :only-of-type:- Selects an element if it is the only one of its type inside the parent.
  • :empty:- Selects elements that have no children (including text nodes).

    • 4. UI/State Pseudo-classes:-

    UI/State pseudo-classes in CSS are used to style elements based on their <strong>user interaction or current state</strong>.
    They are mostly applied to form fields and interactive elements when they are active, hovered, or being interacted with. UI/State Pseudo-classes & Their Meaning:-

  • :hover:- Selects an element when the mouse pointer is over it.
  • :active:- Selects an element when it is being clicked or activated.
  • :focus:- Selects an element (like input or button) when it is focused (clicked or typing inside).
  • :focus-within:- Selects a parent element if any of its child elements is focused.
  • :focus-visible:- Selects an element when it receives focus, but only if it should be visibly indicated (like with keyboard navigation).
  • :target:- Selects an element whose ID matches the URL fragment (e.g., #section in URL).
